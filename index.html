Prework Study Guide

HTML

  • HTML is short for Hypertext Markup Language and is used to create the structure of a webpage.
  • The !DOCTYPE html declaration let's the browser know that the file being opened is an HTML file. Unlike HTML elements, the !DOCTYPE html delaration is not case-sensitive.
  • The head element contains metadata about a webpage. The metadata usually can't be seen unless you use an element inspection tool in a browser.
  • The body element contains content shown to the user.
  • The title element defines the title for a webpage.
  • The style element defines CSS styles for a webpage.
  • The base element defines a webpage's base Uniform Resource Locator (URL).
  • The link element connects an external link to an HTML document.
  • The meta element defines metadata for a webpage, such as any keywords and the viewport.
  • The script element embeds client scripts in an HTML document.
  • The carset element sets the character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for a webpage.
  • The div and span elements both create divisions in a webpage.
  • The heading elements (ranging from h1 to h6) are used to give different sections to a webpage.
  • The p element represents a paragraph or block of text on a webpage.
  • The ul and ol elements are used to show unordered lists and ordered lists, respectively.
  • The li element represents any item listed on an unordered or ordered list.
  • The img element contains information about any images a webpage may have.
  • The src attribute defines the location of an image and is located in the img element.
  • The br element creates a blank line of code in an HTML document.
  • The a element (also known as the anchor element) creates links either to other pages on a website or to a different website.
  • The href attribute contains the URL for a link and is found in the anchor element.
  • The main element shows the dominant content for the HTML document.
  • The footer element is located at the bottom of an HTML document and contains the author, copyright information, contact information, and the sitemap.

CSS

  • CSS is short for Cascading Style Sheets and is used to define the appearance of a webpage.
  • Inline CSS applies to a specific HTML tag with a style attribute that only applies to that particular page element.
  • An internal style sheet is CSS code found directly in an HTML file and is found in the head section.
  • An external style sheet is CSS code found in a separate file that is linked in an HTML file's head. This is the most common way to style webpages.
  • A CSS delcaration contains both a property and a value. This tells a browser how a webpage will be stylized.
  • A margin is the amount of space outside of an element.
  • Padding is the amount of space inside an element.
  • text-align is used to align text in an HTML file.
  • font-size sets the size of the font used in an HTML file.

Git

  • The git command precedes every Git statement.
  • The main branch is the default branch for a GitHub repository.
  • git config lets you change the name and/or email address that is associated with your Git commits.
  • git revert reverts a Git project to a previous version if something goes wrong. Make sure to include the first seven numbers of the SHA (hash) of the version your want to revert to.
  • git status lets you check which branch you are currently on.
  • git checkout lets you move from one branch to another.
  • git chechout -b [branch name] creates a new branch.
  • feature/[branch name] indicates that a branch will be used to work on a feature.
  • git add adds modifications in the current branch to the staging area.
  • git add -A adds all changes made to a branch.
  • git commit -m allows you to make a commit, or change, to a branch. You can also add a brief description of the changes made for the branch.
  • The origin keyword indicates the source for the information that will be pulled.
  • git pull origin [branch name] retrieves a branch's modifications from a remote branch to a local environment.
  • git push origin [branch name] either creates a new branch or adds new changes to an existing branch.
